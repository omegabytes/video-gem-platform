A pocket-sized firmware synth: sixteen knobs, sixteen keys, HDMI out.

Video Gem is not a phone app. It is code that runs on an Adafruit Feather RP2040 DVI and paints 320×240 frames sixty times a second.

The heartbeat: loop()

Arduino-style programs repeat one function forever. Every pass, the board reads controls, updates math, and draws.

CODE 
void loop() {
  keypad.tick();
  handleKeypad();
  updateK13Gesture();

  if (millis() > nextPotUpdateTime) {
    updatePots();
    nextPotUpdateTime = millis() + potIntervalMS;
  }
PLAIN ENGLISH

The main loop runs over and over — this is the rhythm of the device.

Ask the keypad hardware what changed since last time.

Turn those changes into preset switches, program switches, FX mode, and so on.

Notice special timing for the info key (hold vs double-tap).

On a schedule, re-read all sixteen potentiometers through the analog mux.

CODE 
ProgEntry* progForSlot(int slot) {
  for (int i = 0; i < g_numPrograms; i++) {
    if (g_programs[i].draw && g_programs[i].slot == slot)
      return &g_programs[i];
  }
  return NULL;
}
PLAIN ENGLISH

Define a function that looks up one program row by slot number.

Loop through every row in the program table.

If this row has a draw function and its slot matches, we found it.

Hand back the address of that table row so callers can use its callbacks.

End of the loop — try the next row.

No matching row: return NULL (nothing to call).

End of the lookup function.

Render hints: three attitudes

RENDER_CLEAR

Wipe to black, then draw fresh — like a clean canvas each frame.

RENDER_TRAIL

Fade the old frame — motion smears and trails (amount from a global knob).

RENDER_PERPIXEL

No automatic clear — the program paints every pixel itself.

CODE 
ProgEntry g_programs[] = {
  PROG_ENTRY(0, prog_basic_shapes),
  PROG_ENTRY(1, prog_symmetry),
  PROG_ENTRY_EX(2, prog_color, NULL, PROG_FLAG_OWNS_GLOBALS),
  PROG_ENTRY(3, prog_inputs),
  PROG_ENTRY(9, prog_bitmaps),
  PROG_ENTRY(11, prog_fx),
};
int g_numPrograms = sizeof(g_programs) / sizeof(g_programs[0]);
PLAIN ENGLISH

Declare the array that holds every bundled program’s function bundle.

Register slot 0 with the prog_basic_shapes_* family.

Register slot 1 with the prog_symmetry_* family.

Register slot 2 with prog_color_* and mark it as owning global knob meaning.

Register slot 3 with the inputs demo program.

Register slot 9 with the bitmaps program.

Register slot 11 with the FX demo program.

Close the initializer — the table is now fixed in memory.

Compute how many rows exist by dividing total struct bytes by one row’s size.

05

Clever patterns worth stealing

Design moves you can ask AI to replicate in other embedded projects.

Table-driven dispatch

Instead of a giant “if slot == 0 … else if slot == 1 …”, the code stores function pointers in a table. Adding a mode is mostly data, not new branching spaghetti.

Global FX without program rewrites

Effects run in the platform pipeline — programs don’t call them. That means a new shader-like trick can ship once and layer on every visual mode.

Palette stage

Twist colors before pixels hit the glass.

Pre-draw

Decay or shift indices before the program paints.

Post-draw

Mirrors and feedback after the program finishes.

CODE 
typedef struct {
  int slot;
  void (*draw)(int preset);
  uint8_t (*renderHint)(int preset);
  const char* (*name)(void);
  const char* (*character)(void);
  const char* (*presetName)(int preset);
  const char* (*potLabel)(int preset, int pot);
  void (*init)(void);    // optional: called on program switch (NULL = no-op)
  uint8_t flags;
} ProgEntry;
PLAIN ENGLISH

Start defining one row’s shape for the program table.

Which program slot number this row occupies.

Pointer to the function that draws one frame for a given preset.

Pointer to the function that returns clear, trail, or per-pixel behavior.

Pointer to the function that returns the program’s display name.

Pointer to a short tagline string for overlays.

Pointer to a function that names each preset by index.

Pointer to a function that labels each pot for hints and overlays.

Optional init hook when switching programs — or NULL for none.

Small flags field (e.g. this program owns the global knobs).

End of the struct type named ProgEntry.

When things smell off

Debugging intuition for hardware + embedded loops.

Symptom → first guess

Upload refuses

Wrong serial port or board in bootloader — run board list, try set-board.

Black screen, LED blinks

display.begin failed — power or wiring to DVI path.

Host tests flaky

Stale .gcda from coverage — clean test/build and rerun.

🧪
Tests run on your laptop

Unit tests mock the framebuffer so logic runs fast. They don’t prove HDMI timing — only that the math and state machines behave.
